01  Overview
ConnectChat is a production-deployed real-time messaging platform built from scratch — covering the full engineering surface: backend API design, WebSocket communication, JWT security, database modelling, containerisation, cloud infrastructure, and automated deployments.

Every layer is production-minded: HTTPS via Nginx, automatic token refresh, database on AWS RDS, CI/CD on every push. This isn't a toy — it's running live, serving real traffic.

02  Features
⚡
Real-Time Messaging
Bidirectional private messaging over WebSocket + STOMP protocol. Messages deliver instantly without polling — true real-time.

JWT Auth Flow
Secure access + refresh token architecture with Spring Security. Automatic token refresh handles session expiry gracefully on the client.
🟢
Presence Tracking
Live online/offline status for every user. Presence updates propagate in real-time via WebSocket — no stale state.

05  Engineering Highlights
01
Real-Time System Design
Designed bidirectional WebSocket communication with STOMP subscriptions for per-user message channels. Handled connection lifecycle, reconnects, and presence heartbeats — ensuring the UI stays consistent without page refreshes.
WebSocketSTOMPSockJS
02
JWT Security Architecture
Implemented a dual-token system — short-lived access tokens for API calls and longer-lived refresh tokens for session continuity. Spring Security filters validate every request; the React client silently refreshes tokens before expiry.
Spring SecurityJWTToken Refresh
03
Containerised Cloud Deployment
Dockerised the Spring Boot backend with a multi-platform build for AWS EC2 (linux/amd64). Nginx handles SSL termination and routes HTTP/WebSocket traffic to the container. Database lives on AWS RDS for managed durability.
DockerAWS EC2NginxRDS
04
Automated CI/CD Pipeline
GitHub Actions fires on every push to main: builds the Docker image, pushes to Docker Hub, SSHs into EC2, and restarts the container — zero manual intervention. The pipeline takes under 3 minutes from push to production.
GitHub ActionsDocker HubSSH Deploy
